# Source Directory Structure

This directory contains the main application code, organized by feature and role:

*   **`assets/`**: Static assets like images, icons, and fonts that are imported directly into components.
*   **`components/auth/`**: React components specifically related to user authentication (e.g., login forms, registration, password reset).
*   **`components/library/`**: Reusable or specific domain-level components that don't fit into generic UI or shared categories.
*   **`components/shared/`**: Generic, highly reusable components used across multiple pages or features (e.g., specific layouts, generic cards, or complex composite base elements beyond simple shadcn UI pieces). 
*   **`components/ui/`**: Base UI components generated by shadcn/ui.
*   **`hooks/`**: Custom React hooks for encapsulating complex state or side effects.
*   **`pages/`**: Top-level page components that represent individual routes in the application.
*   **`utils/`**: Helper functions, constants, and utilities that don't depend on React.
